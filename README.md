# Autonomous Report Agent (Olist Data Warehouse)

## Goal of this Project
The goal of the Report Agent is to automate the process of business intelligence reporting. By interpreting natural language prompts from users, the agent queries the Olist E-commerce Data Warehouse (stored in DuckDB), analyzes the data, generates visualizations, and synthesizes actionable insights into a professional markdown report. It acts as an autonomous data team capable of answering both simple lookups and complex strategic business questions.

## Architecture
This project implements a multi-agent system using **CrewAI**. The workflow consists of three specialized AI agents working sequentially:
1. **Router & Strategy Agent (Prompt Evaluator):** 
   Reads the knowledge base (`business_goal.md` and `DATA_DICTIONARY.md`) to understand the business context. It evaluates the user's prompt and breaks complex questions down into an actionable, step-by-step SQL analysis plan.
2. **Senior Data Analyst:** 
   Receives the plan and interacts directly with the DuckDB Data Warehouse. It intelligently decides whether to query pre-aggregated Mart layers (for speed and consistency) or drill down into Staging/Intermediate layers to extract the necessary numerical facts and dimensions.
3. **Data Storyteller & Visualizer:** 
   Takes the raw SQL results and crafts a cohesive narrative adhering to the requested tone and language. It utilizes Python to dynamically generate data visualizations (`matplotlib`/`seaborn`) and saves the final polished Markdown report alongside its visual assets in a dedicated, timestamped folder.

## Techstack
- **Orchestration Framework:** CrewAI
- **Language Models:** Gemini API (Google GenAI) via LiteLLM wrapper
- **Database:** DuckDB (fast, local analytical database)
- **Programming Language:** Python 3
- **Data Manipulation & Visualization:** `pandas`, `matplotlib`, `seaborn`
- **Tooling Integrations:** `crewai_tools` (File/Directory readers, Python Execution Tool, Custom DuckDB Query Tool)

## How to Use
1. **Environment Setup:** Ensure your `.env` file at the root contains your valid `GEMINI_API_KEY`.
2. **Dependencies:** Activate your Python virtual environment and ensure all necessary libraries (`crewai`, `duckdb`, `pandas`, `matplotlib`, `seaborn`, `python-dotenv`) are installed.
3. **Run the Agent:**
   Open a terminal in the `report_agent` directory and execute `main.py` along with your analytical question. The agent handles natural language requests in English or Vietnamese.
   ```bash
   cd d:\src\report_agent
   python main.py "Phân tích hoạt động kinh doanh của seller 53243585a1d6dc2643021fd1853d8905 trong năm 2017. Phân tích những drivers ảnh hưởng tới kết quả kinh doanh. Đưa ra những khía cạnh cần cải thiện cho sellers này?, Báo cáo bằng Tiếng Anh"
   ```
4. **View Output:** The agent will dynamically create a unique, timestamped folder under `results/` (e.g., `results/report_20260506_161500/`) containing:
   - `final_report.md`: The complete narrative, insights, and recommendations.
   - `*.png`: Any charts or graphs autonomously generated by the Data Storyteller.

## Expected Value to Business
- **Time-to-Insight Reduction:** Reduces the time required to generate ad-hoc reports from days (or hours) to mere minutes.
- **Accessibility for Stakeholders:** Enables non-technical executives and managers to interact with the data warehouse using plain natural language without needing SQL skills.
- **Data-Driven Decisions:** Provides actionable recommendations based directly on accurate, up-to-date data, aligning closely with the company's OKRs and business goals.
- **Consistency & Standardization:** By enforcing "Tone of Voice" and "Language" parameters in the agent's prompt, the generated reports are highly professional and ready for executive presentation.

## Limitation and Future Enhancement
### Limitations
- **API Constraints & Overloads:** Relies heavily on the Google Gemini API. It is subject to rate limits and server overloads (e.g., HTTP 503/429). This is currently mitigated using Exponential Backoff Retries, but severe API outages will halt the agent.
- **Context Window Constraints:** Extremely large SQL outputs (e.g., returning millions of rows directly to the LLM instead of aggregating them first) can exceed the LLM's context window. The agent relies heavily on the Data Analyst performing proper `GROUP BY` aggregations.
- **Autonomous Code Execution:** The agent generates and executes Python code autonomously for visualizations. While powerful, it can occasionally produce syntax errors or formatting issues if the data shape is highly irregular.

### Future Enhancements
- **Human-in-the-Loop (HITL):** Introduce a validation step where the Strategy Agent presents the SQL execution plan for human approval before actually querying the database, preventing expensive or incorrect queries.
- **Self-Correction & Reflexion:** Allow the Data Storyteller to read back its own generated charts using Vision models, ensuring it can retry and fix the code if the chart is unreadable or fails to render.
- **Interactive Dashboards:** Extend the output formats beyond static Markdown to generate complete HTML/React dashboards (e.g., using Streamlit) for interactive drill-downs.
- **Long-Term Memory Integration:** Implement memory (via CrewAI's memory system) so the agent remembers previous reports, organizational context, and user preferences across multiple sessions.
- **Local Model Support (Ollama):** Use local models through Ollama to ensure data security and cost efficiency, reducing reliance on external APIs.
- **Semantic Layer & Enhanced Metadata:** Develop a more comprehensive semantic layer that includes complex business logic (e.g., LTV definitions, cohort logic) and granular metadata to improve the accuracy of the agent's SQL generation and reasoning.
